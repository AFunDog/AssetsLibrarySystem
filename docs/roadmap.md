# 未来计划

`docs/` 只记录尚未实现或仍需继续完善的方案，不承担项目现状、运行方式、代码结构或接口说明。

项目现状与使用方式以以下文档为准：

- 根目录 `README.md`：项目整体说明
- `src/avalonia/README.md`：桌面端、Application 与 Console 说明
- `src/backend/README.md`：Python 模型网关说明
- 各级 `AGENTS.md`、`CLAUDE.md`：智能体开发约束

## 近期计划

### 检索质量

- 将多角度向量从“按素材取最高分角度”升级为可评测的角度加权聚合。
- 为全面、主体、场景、风格、情感、乐器等角度建立稳定权重与兼容策略。
- 在搜索结果中展示命中角度和关键片段，提高结果可解释性。
- 建立覆盖文本、图片、视频、音频的固定检索评测集。

### 素材生命周期

- 完善 `asset_uid` 冲突、内容变化、文件移动和副本识别策略。
- 持久化批量描述、向量化、索引重建等后台任务状态。
- 增加可恢复的批处理与失败重试能力。

### 描述与向量维护

- 扩大图片、视频和文本素材的结构化描述覆盖率。
- 支持按内容指纹和模型版本识别过期描述及过期向量。
- 优化批量向量化接口和索引增量更新流程。

## 中长期计划

### 产品化

- 增加检索质量、模型耗时、token 使用量和失败原因统计。
- 完善设置页中的提示词管理与真实模型调用联动。

## 暂不计划

- 引入独立向量数据库、消息队列或复杂分布式部署。
- 在 Python 后端重复实现素材库、目录扫描和本地元数据管理。
- 在没有评测结果前引入复杂动态权重或自动调参系统。
- 实现 RAG、知识问答、答案生成或检索上下文拼装。
